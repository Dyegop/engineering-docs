------ HTTP BASICS ------

HTTP (Hypertext Transfer Protocol) is a protocol used by web browsers and servers to communicate.
It is stateless, text-based, and follows a request-response model.


HTTP Request/Response Syntax:

GET /users?id=123 HTTP/1.1
Host: api.example.com
Authorization: Bearer <token>
Content-Type: application/json

{ "name": "Test" }


HTTP/1.1 200 OK
Content-Type: application/json

{ "id": 123, "name": "Diego" }


Method: GET, POST, etc.
Path/URL: /users?id=123
Version: HTTP/1.1
Headers: Metadata (host, authorization, content-type, user-agent, etc)
Body: Payload (used in POST/PUT)




------ HTTP METHODS ------

| Method    | Use Case                              | Body Allowed | Idempotent | Safe  |
| --------- | ------------------------------------- | ------------ | ---------- | ----- |
| GET       | Retrieve a resource                   | No           | Yes        | Yes |
| POST      | Create a new resource                 | Yes          | No         | No  |
| PUT       | Replace a resource (full update)      | Yes          | Yes        | No  |
| PATCH     | Partially update a resource           | Yes          | Yes        | No  |
| DELETE    | Delete a resource                     | Sometimes    | Yes        | No  |
| HEAD      | Same as GET, no response body         | No           | Yes        | Yes |
| OPTIONS   | Discover supported HTTP methods       | No           | Yes        | Yes |
| CONNECT   | Establish a tunnel (e.g., for HTTPS)  | Rarely       | Yes        | No  |
| TRACE     | Echo the received request (debugging) | No           | Yes        | Yes |




------ STATUS CODES ------

1xx ‚Äî Informational

| Code | Meaning                         | Description                                         |
| ---- | ------------------------------- | --------------------------------------------------- |
| 100  | Continue                        | Client should continue request                      |
| 101  | Switching Protocols             | Server is switching to a different protocol         |
| 102  | Processing (WebDAV)             | Server has received and is processing the request   |
| 103  | Early Hints                     | Used to preload resources before final response     |


2xx ‚Äî Success

| Code | Meaning                         | Description                                         |
| ---- | ------------------------------- | --------------------------------------------------- |
| 200  | OK                              | Request succeeded                                   |
| 201  | Created                         | New resource created                                |
| 202  | Accepted                        | Request accepted but processing not complete        |
| 203  | Non-Authoritative Info          | Metadata may differ from origin server              |
| 204  | No Content                      | Success, but no content returned                    |
| 205  | Reset Content                   | Client should reset form/view                       |
| 206  | Partial Content                 | Partial response (e.g., video streams)              |
| 207  | Multi-Status (WebDAV)           | Multiple responses for batch operations             |
| 208  | Already Reported                | Resource previously reported                        |
| 226  | IM Used                         | Response is result of instance-manipulations        |


3xx ‚Äî Redirection

| Code | Meaning                         | Description                                         |
| ---- | ------------------------------- | --------------------------------------------------- |
| 300  | Multiple Choices                | More than one resource found                        |
| 301  | Moved Permanently               | Resource moved to new URL                           |
| 302  | Found (Temporary)               | Temporary redirect                                  |
| 303  | See Other                       | Redirect using GET method                           |
| 304  | Not Modified                    | Cached version is valid                             |
| 305  | Use Proxy (deprecated)          | Use a proxy to access resource                      |
| 307  | Temporary Redirect              | Same method must be used on redirect                |
| 308  | Permanent Redirect              | Like 301, but keeps method/headers                  |
